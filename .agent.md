### Canonical Data Semantics

- `Email_Source` values: `people_api` = verified from Google People API; `manual_verified` = user manually confirmed the email and reconciled it in the workflow; `guessed` = deterministic guess, not verified; `ambiguous` = unresolved People API result and must be treated like guessed; blank = unset and should be treated like guessed until verified.
- `Confidence_Level` values: `very_high`, `high`, `medium`, `low`, `unconfirmed`. Blank means no research confidence has been written yet. Never write email-source labels such as `people_api` or `manual_verified` into `Confidence_Level`.
- `STATUS` values: `PENDING` = research not started; `PROCESSING` = actively in the async Phase 2 sheet pipeline; `RESEARCH_DONE` = profile is ready and email generation is next; `EMAIL_DONE` = email content is ready for send; `SENT` = Apps Script already sent it; `FAILED` = processing failed and needs retry; `FAILED_PARSE` = stored output was malformed and was cleared; `SKIP_GUESSED_EMAIL` = terminal manual-workflow block for guessed/blank email rows.
- Apps Script send gate: a row is send-eligible only when `Email_Source` is `people_api` or `manual_verified` and `STATUS` is exactly `EMAIL_DONE`. Verified source alone is insufficient, and `EMAIL_DONE` with a guessed/ambiguous source must not send.
- `reconcile-email`: sets `Email_Source=manual_verified`, applies `--status` with default `PENDING`, accepts only `PENDING`, `RESEARCH_DONE`, or `SKIP_GUESSED_EMAIL`, and appends a timestamped entry to `manual_workflow/manual_overrides.md`.

## Architecture (3 Phases)

### Phase 1 — Contact Resolution (`phase1_contact_resolution.py`)
- Input: `iimu_2025.csv` (AlmaConnect export, 80 alumni) → Google People API → Google Sheet
- Resolves `@iimu.ac.in` emails; falls back to `firstname.lastname.YEAR@iimu.ac.in` guess
- Auto-routes by filename to `cohort_{year}` when the CSV stem contains one or more `20xx` years; the last match wins unless `--tab TAB` is supplied.
- Commands: `python main.py phase1 iimu_2025.csv`, `python main.py phase1 --tab cohort_2015 batch_csv/iimu_2015.csv`, `python main.py phase1-retry [--tab TAB]`

### Phase 2 — LLM Research + Email Generation (`phase2_orchestrator.py`)
- **2A** (`phase2a_alumni_research.py`): [FROZEN PATH] No pre-fetch. User message = `{name} {year} passout` only. Phase 2A routes Gemini research through the native `generateContent` REST endpoint with `google_search` grounding when `GEMINI_SEARCH_GROUNDING=true`. Returns `<PROFILE>` XML block. Tavily code preserved but bypassed via `config.TAVILY_FROZEN`.
- **2B** (`phase2b_email_generation.py`): Generates `<EMAIL>` XML block gated by confidence. Anti-hallucination rules enforced.
- Sheet mode uses bounded async concurrency through `_run_from_sheet_async()` with Gemini/Tavily semaphores, config-driven concurrency limits, and lock-protected cache mutation.
- Two modes: `json` (local file I/O) or `sheet` (reads/writes Google Sheet)
- Commands: `python main.py phase2-sheet 3` (test 3 rows) / `python main.py research "Name"`

### Phase 3 — Apps Script Sending (`apps_script_sender.js`)
- Runs inside Google Sheet's Apps Script editor (not Python)
- Batch sends are now tab-aware (`sendBatch(tabName)`) and multi-cohort aware (`sendAllPendingCohorts()`).
- Sending respects `GMAIL_QUOTA_LIMIT` and `GMAIL_QUOTA_BUFFER`, and only sends rows where `Email_Source` is verified (`people_api` or `manual_verified`) and `STATUS == EMAIL_DONE`.
- Commands: `previewNextBatch("cohort_2015")` → `checkQuota()` → `setupTrigger()`

### Manual Workflow — Semi-Manual LLM Pipeline (`manual_workflow/manual_workflow.py`)
- Exports PENDING rows from a cohort tab into `research_queue.md` as batches of 5 with RECONCILE_IDs.
- User pastes batches into an external LLM with system prompts, then pastes `<PROFILE>` and `<EMAIL>` outputs into `ingest_queue.md` / `review_queue.md`.
- Ingest reconciles results back to the Google Sheet using 3-tier RECONCILE_ID matching (verified_email > guessed_email > name_year).
- Ingest now pre-deduplicates repeated RECONCILE_IDs inside one paste, normalizes dirty sheet-side confidence before comparison, resets stale `EMAIL_DONE` email content when a profile upgrade wins, writes `SKIP_GUESSED_EMAIL` for guessed/blank email sources, and batches all row writes before queue cleanup.
- Successful ingest rewrites `ingest_queue.md` back to the clean template with a `_Last ingested:` header instead of leaving old `INGEST_RUN_COMPLETE` markers in place.
- Successful ingest now auto-refreshes `email_queue.md` so rows promoted back to `RESEARCH_DONE` reappear for manual email generation without a separate export step.
- Review queue handling is now first-class through the CLI, and `status --detail cohort_YYYY` gives a terminal-side row view without opening Google Sheets.
- Commands:
  - `python manual_workflow/manual_workflow.py export --tab cohort_2018 --batch-size 5` — export PENDING rows to `research_queue.md`
  - `python manual_workflow/manual_workflow.py ingest` — process new `<PROFILE>` and `<EMAIL>` blocks from `ingest_queue.md`
  - `python manual_workflow/manual_workflow.py ingest --force` — ignore old ingest markers and re-process all blocks
  - `python manual_workflow/manual_workflow.py review --list` — inspect review_queue.md grouped by reason
  - `python manual_workflow/manual_workflow.py review --retry "Aditya Sharma"` — move one review item back to `ingest_queue.md`
  - `python manual_workflow/manual_workflow.py status --detail cohort_2018` — show actionable row detail for one tab
  - `python manual_workflow/manual_workflow.py rebatch --batch-size 5` — reorganize remaining batches in `research_queue.md`

## Coding Conventions

- **LLM provider:** Use the `MODEL_PROVIDER` abstraction in `config.py`. Do not hardcode provider URLs, model strings, or API keys outside `config.py`.
- **Provider quirks:** Route request kwargs through `config._sanitize_kwargs(...)` before every Phase 2 model call.
- **Config:** All constants and env vars in `config.py`. No hardcoded secrets anywhere.
- **Sender identity:** Edit `prompts/sender_profile.txt` for sender biography/context; do not reintroduce inline sender strings in `config.py`.
- **Concurrency:** Use `config.GEMINI_CONCURRENCY` and `config.TAVILY_CONCURRENCY`; do not reintroduce hardcoded semaphore counts.
- **Structured output:** Phase 2A returns `<PROFILE>` XML. Phase 2B returns `<EMAIL>` XML. Keep parsers strict; the only accepted email fallback is the narrow missing-`BODY:` recovery for otherwise valid `<EMAIL>` blocks.
- **Anti-hallucination:** Emails reference only facts verified at or above confidence threshold. Never fabricate career details.
- **Confidence gating:** Follow the canonical `Confidence_Level` values defined in the Current State section. Each level gates which hooks may appear in email.
- **Error handling:** Graceful degradation — if research fails, fall back to generic alumni template.
- **Retry behavior:** HTTP 401/403 auth failures are unrecoverable in `utils/retry.py` and should fail immediately without retry.
- **Tavily key policy:** Only credit exhaustion rotates to the next Tavily key. Other Tavily errors should propagate.
- **Checkpoint semantics:** Resume state now stores the next cache index, not a visible row number.
- **Sheet initialization:** In Phase 2 cohort/sheet flows, missing tabs should raise rather than being auto-created.
- **Concurrency:** In sheet mode, preserve the bounded async orchestration path rather than reintroducing ad-hoc sleeps or unbounded tasks.
- **Live batch execution:** Always pipe live runs to disk with `Tee-Object`. Never start an unpiped `phase2-sheet` batch.
- **Prompt files are READ-ONLY for agents.** Never edit `prompts/*.txt`, `manual_workflow/system_prompt_research.md`, or `manual_workflow/system_prompt_email.md`. Suggestions are allowed; direct edits are forbidden.
- **Manual workflow terminal status rule:** `SKIP_GUESSED_EMAIL` is terminal in the manual ingest path and must not be overwritten by new manual-ingest code paths.

## Data Integrity & Quality Safeguards

- **Flaw #4 (partial data cleanup):** When a `RESEARCH_DONE` row fails to parse its stored `<PROFILE>`, the orchestrator now clears `Verified_Company`, `Enrichment_Notes`, and `Enrichment_Source` atomically and sets `STATUS=FAILED_PARSE`. The broken XML is logged (first 300 chars) before clearing.
- **Flaw #5 (placeholder validation):** `phase2b_email_generation.py` checks generated subject+body for bracket-style template placeholders using a strict allowlist matcher. If found, the email is rejected (returns `None, None`) and logged as `PLACEHOLDER_DETECTED`.
- **Flaw #8 (Tavily exhaustion):** When all Tavily keys are exhausted mid-batch, remaining rows are now marked `PENDING` (retry-friendly) instead of `FAILED`. The count of affected rows is logged.
- **Tavily run-state safety:** `_EXHAUSTED_TAVILY_KEYS` is cleared at the start of each Phase 2 run, and non-credit Tavily failures no longer rotate across keys.
- **Flaw #9 (duplicate detection):** `utils/deduplication.py` provides `detect_duplicates()` (exact email) and `detect_fuzzy_name_duplicates()` (SequenceMatcher, threshold 0.85). Phase 2 sheet mode now runs write-through duplicate preflight before processing, and `fix_duplicates.py` applies the same logic manually.
- **Duplicate decision rule:** Different non-empty emails are treated as confirmed different people even if company data is blank. Company comparison only applies after email identity fails to resolve a fuzzy-name pair.
- **Tavily raw data logging:** `Tavily_Raw` and `Tavily_Metadata` columns persist raw search results to the sheet. Raw LLM responses are logged to `debug_logs/debug_{run_id}.jsonl` (capped at 10K chars per entry).
- **Checkpoint safety:** `progress.json` now stores the next cache index to resume from. Older checkpoint files are incompatible with the new semantics.
- **Sheet-cache safety:** Shared async sheet tasks now serialize in-memory cache mutation with `SheetCache._lock`.

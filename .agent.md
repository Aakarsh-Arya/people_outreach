## Architecture (3 Phases)

### Phase 1 — Contact Resolution (`phase1_contact_resolution.py`)
- Input: `iimu_2025.csv` (AlmaConnect export, 80 alumni) → Google People API → Google Sheet
- Resolves `@iimu.ac.in` emails; falls back to `firstname.lastname.YEAR@iimu.ac.in` guess
- Commands: `python main.py phase1 iimu_2025.csv` / `python main.py phase1-retry`

### Phase 2 — LLM Research + Email Generation (`phase2_orchestrator.py`)
- **2A** (`phase2a_alumni_research.py`): Tavily web search → Gemini 2.5 Flash research agent → structured `<PROFILE>` XML block with confidence scoring
- **2B** (`phase2b_email_generation.py`): Generates `<EMAIL>` XML block gated by confidence. Anti-hallucination rules enforced.
- Two modes: `json` (local file I/O) or `sheet` (reads/writes Google Sheet)
- Commands: `python main.py phase2-sheet 3` (test 3 rows) / `python main.py research "Name"`

### Phase 3 — Apps Script Sending (`apps_script_sender.js`)
- Runs inside Google Sheet's Apps Script editor (not Python)
- Batch sends 5 emails every 5 minutes with random jitter. Quota-guarded.
- Commands: `previewNextBatch()` → `setupTrigger()`

## Coding Conventions

- **LLM provider:** Always use Google AI Studio via the `openai` library with `base_url` override. Model: `gemini-2.5-flash`.
- **No extra_body / non-standard params** — the OpenAI client bridge breaks on them. Only pass standard parameters.
- **Config:** All constants and env vars in `config.py`. No hardcoded secrets anywhere.
- **Structured output:** Phase 2A returns `<PROFILE>` XML. Phase 2B returns `<EMAIL>` XML. Use strict fenced parsers — no regex fallbacks.
- **Anti-hallucination:** Emails reference only facts verified at or above confidence threshold. Never fabricate career details.
- **Confidence levels:** `very_high` > `high` > `medium` > `low` > `unconfirmed`. Each level gates which hooks appear in email.
- **Error handling:** Graceful degradation — if research fails, fall back to generic alumni template.
- **Rate limiting:** Respect `PEOPLE_API_DELAY` (0.3s) and `ENRICHMENT_DELAY` (1.0s) between API calls.
